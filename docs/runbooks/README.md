# Runbooks

What to do when something breaks. One file per scenario.

A useful runbook:

- Starts with an observable symptom ("the API returns 500 on /foo").
- Lists concrete steps to diagnose.
- Explains how to mitigate (even temporarily).
- Includes exact commands, not vague descriptions.

## When to write a runbook

The first time something breaks and you fix it, write the runbook. The second time it happens, your future self (or an agent) will thank you.
